# Story Presentation Plan

## Core story
Akash should not read like a generic "software engineer portfolio."
He should read like a high-agency technical generalist: builder, researcher, and operator.

The site should communicate:
- he works across AI, robotics, math, and product
- he likes systems under pressure and real constraints
- he turns messy inputs into usable, visual, high-leverage outputs
- he is young, ambitious, and already shipping things with real-world texture

## Research takeaways
From portfolio/storytelling references, the strongest repeated patterns were:

1. **Lead with a visual identity, not a paragraph dump**
   - strongest sites open with a bold name, compact framing, and rich visual context
   - text should explain *after* attention is earned

2. **Use motion as pacing, not decoration**
   - subtle reveal, hover, parallax-ish depth, and scroll pacing work
   - loud novelty animation hurts credibility fast
   - animation should guide hierarchy and create rhythm

3. **Show a point of view through project selection**
   - the first few projects define the person
   - the homepage should intentionally frame the "signal projects"

4. **Narrative beats matter more than section count**
   - identity
   - signal / proof
   - worldview
   - depth
   - contact / next step

5. **Case studies need visual anchors**
   - big image/video first
   - then concise framing: problem, system, outcome, role
   - avoid giant uninterrupted markdown blocks near the top

6. **Mobile should preserve the same emotional impression**
   - not just responsive layout
   - hero still needs impact
   - tiles should still feel dense and intentional
   - important CTAs must stay visible early

## Design direction for Akash specifically
Blend three vibes:
- **editorial**: strong typography, contrast, pacing
- **systems minimal**: clean structure, technical calm, no clutter
- **student builder energy**: motion, demos, campus/product momentum

## Homepage story order
1. Hero: name, face, motion, short framing line
2. Signal projects: Scarlet Sync, Lykke, Rutgers Economics Labs, Robotics
3. Selected work grid: stronger proof with click-through depth
4. Optional deeper strip later: writing / mindset / operating principles

## About page story
Should explain the through-line:
- not random interests
- one person who likes turning complexity into clear systems
- research + product + robotics all point to the same taste

## Project page template target
Each project page should trend toward:
- hero media
- one-line why it matters
- quick metadata / role / stack / impact
- main narrative
- media breaks

## Motion rules
- use reveal transitions, hover lift, soft scale, layered depth
- respect prefers-reduced-motion
- keep interactions smooth and quiet
- avoid gimmicky cursor tricks / overloaded parallax / chaotic autoplay

## Mobile rules
- keep hero copy to 2-3 visual beats max
- preserve image density above the fold
- keep tap targets obvious
- avoid long stacked paragraphs early
- use cards and strong spacing instead of walls of text
